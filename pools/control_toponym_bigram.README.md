# control_toponym_bigram — phonotactic control pool

Generated by `scripts/build_control_pools.py --sampler bigram` from `pools/toponym.yaml`. Created for **mg-9f18 / harness v18** to address the v10 toponym-pool gate failure: the v6 unigram sampler produced control surfaces like `eoao` and `aathei` that the Basque LM scored extremely well by raw phoneme-frequency match. The bigram-preserving sampler draws each phoneme conditional on the previous phoneme, so the realized control surfaces inherit the substrate's adjacent-phoneme structure rather than just its marginal histogram.

## Matching algorithm

1. **Inventory.** Compute the phoneme-frequency histogram and the bigram-frequency map (one Counter per `prev_phoneme`) across all entries in `pools/toponym.yaml`.
2. **Length distribution.** For each substrate entry of length L, emit exactly one control entry of length L. The control pool is therefore byte-equal in entry count and length distribution to the substrate pool.
3. **First-phoneme sampling.** The first phoneme of each control surface is drawn from the substrate's *positional-onset* unigram distribution (i.e. counts of phonemes at position 0 in the substrate's `phonemes[]` lists), Laplace-smoothed by alpha=0.1: `weight(p) = onset_count(p) + alpha`. Phonemes that never appear at position 0 in the substrate still receive non-zero probability, so the sampler can produce any inventory phoneme as an onset.
4. **Subsequent-phoneme sampling.** Each subsequent phoneme c is drawn conditional on the previous phoneme `prev` with weight `bigram_count(prev, c) + alpha`. For phonemes that appear only word-finally in the substrate (so `prev` has no observed successors), the sampler falls back to the onset distribution from step 3 — this is rare in practice but keeps the draw well-defined.
5. **Surface.** The control surface is the concatenation of the sampled phonemes — purely orthographic, no semantic content. Within-pool uniqueness is enforced by redrawing collisions up to 50 times; if uniqueness still fails the RNG seed is bumped deterministically (so the output remains reproducible).
6. **Region tag.** Every control entry carries `region: phonotactic_control_toponym_bigram`. This is deliberately unmapped in `harness/metrics._GG1_REGION_COMPAT`, so geographic_genre_fit_v1 falls back to neutral 0.5 across the whole control pool — controls have no semantic content and should not be flattered by the geographic prior.
7. **Semantic field.** Unset on every control entry (gg1 also falls back to 0.5).
8. **Determinism.** Seed = `0x1b7b5d4ef69cede7` (sha256("control_pool:toponym:bigram")[:16]). The bigram-sampler key is suffixed with `:bigram` so this pool draws from a disjoint random stream relative to the v6 unigram control. Re-running the builder produces a byte-identical YAML. Asserted by `harness/tests/test_control_pools.py`.

## What the control is NOT

- Not a *Linear-A surface* control. The substrate pool's surfaces are real lexemes from a real attested language; control surfaces are synthetic concatenations with no semantic content.
- Not a *trigram* (or higher-order) control. Adjacent-phoneme structure is matched, but three-phoneme spans and longer-range dependencies are not preserved. v18's design choice was to address the most prominent unigram-sampler artifact (control surfaces with extreme phonotactic violations) without absorbing the substrate signal entirely; matching trigrams + longer would encode the substrate's word-shape statistics so completely that any LM trained on related material would fail to discriminate.
- Not a *position* control beyond first-phoneme onset weighting. Mid-word positional dependencies (e.g. `r` is more likely than `d` to appear in the final position) are partially captured by the bigram transitions but not enforced as a separate term.

What the control DOES match: phoneme inventory, length distribution, and adjacent-phoneme transitions. The null hypothesis it lets us test is *"the substrate per-surface leaderboard is just an artifact of phoneme-inventory + bigram-phonotactics overlap with the syllabogram set"* — strictly stronger than the unigram null v6 tested.
